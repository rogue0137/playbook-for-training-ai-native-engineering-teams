# No Vibe Coding Checklist

For every PR:

- [ ] PR is within size guidelines (e.g., under ~500 changed lines or team-agreed threshold).
- [ ] I wrote the description myself before using AI.
- [ ] Description includes:
  - [ ] Intent (what outcome I'm aiming for)
  - [ ] Scope (what changed and what didn't)
  - [ ] Risk (where things could break)
- [ ] I requested an AI review on the PR.
- [ ] I read the AI review and compared it to my own description.
- [ ] If AI surfaced something important, I either:
  - [ ] Updated the code, or
  - [ ] Updated the description, or
  - [ ] Added a comment explaining the decision.
