# No Vibe Coding Reviewer Checklist

When reviewing a PR (as a human reviewer):

- [ ] PR size is reasonable and focused.
- [ ] Human description:
  - [ ] Explains the “why,” not just the “what.”
  - [ ] Matches the actual changes in the diff.
- [ ] AI review was run (if available for this repo).
- [ ] AI review didn’t surface any unaddressed red flags.
- [ ] Any AI-suggested changes were either:
  - [ ] Applied and tested, or
  - [ ] Intentionally rejected with a clear reason.

Optional reflection:

- Did the AI review help you review faster or see something you might have missed?
- Did it create noise or overconfidence?
